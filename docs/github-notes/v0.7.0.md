# v0.7.0 — Popular Packages Corpus + Polish + QA (FASE 6)

## Changes

### Popular Packages Corpus

- **`scripts/fetch_popular_packages.py`** — New CLI script to download the top 5000 most popular packages from PyPI and npm with weekly download counts
  - PyPI: Uses [hugovk/top-pypi-packages](https://hugovk.github.io/top-pypi-packages/) API (updated monthly)
  - npm: Triple strategy — search API with popularity sorting + curated seed list (~170 packages) + bulk downloads API for real counts
  - Supports `--top N`, `--pypi-only`, `--npm-only`, `--output-dir`, `--timeout`
  - Rate limiting: 200ms between search queries, 100ms between download batches
- **`data/popular_pypi.json`** — 5000 PyPI packages with weekly download counts (~140 KB)
- **`data/popular_npm.json`** — 3454 npm packages with weekly download counts (~121 KB)
- **`data/placeholder_patterns.json`** — 30 placeholder patterns reference file
- DEP-003 (typosquatting) now uses the real-world corpus of 8400+ packages instead of the ~300-package builtin fallback, significantly improving detection accuracy

### CLI Polish

- **Output file error handling**: `vigil scan --output report.json` now catches `OSError` (read-only directory, disk full) with a clear error message to stderr and exit code 2. Previously crashed with an unhandled exception.
- All CLI flags verified and tested: `--quiet`, `--verbose`, `--offline`, `--changed-only`, `--format`, `--category`, `--rule`, `--exclude-rule`, `--language`, `--fail-on`, `--output`, `--config`

### Bug Fixes

- **httpx response outside context manager**: `resp.json()` in `fetch_pypi_top()` was called after the `with httpx.Client` block exited — moved inside the context manager
- **Duplicate npm seed packages**: Removed duplicate entries for "nuxt", "yargs", and "commander" in the npm seed list
- **Missing JSONDecodeError handling**: `resp.json()` calls in npm search, bulk download, and scoped download now catch `json.JSONDecodeError`
