# v1.0.0 — Stable Release

The first stable release of vigil. All 4 analyzers fully implemented, exhaustive QA with 164 manual tests across 18 phases, 10 bugs fixed, 1706 automated tests passing at 97% coverage.

## Highlights

- **4 analyzers active**: Dependency (DEP), Auth (AUTH), Secrets (SEC), Test Quality (TEST)
- **24 of 26 rules implemented** across 4 categories
- **4 output formats**: human (terminal), JSON, JUnit XML, SARIF 2.1.0
- **8400+ popular packages corpus** for typosquatting detection (5000 PyPI + 3454 npm)
- **Exhaustive QA**: 164 manual tests in 18 phases covering every subcommand, format, flag, rule, and edge case
- **1706 automated tests**, 97% coverage

## What's New Since v0.7.0

### Bug Fixes (V1 QA)

10 bugs found during exhaustive QA testing, all fixed:

- **Damerau-Levenshtein for typosquatting** (BUG-001): Transposition typos like `reqeusts` → `requests` are now detected. Previously required 2 edits (Levenshtein), now correctly counted as 1 (Damerau-Levenshtein).
- **`vigil tests --offline` flag** (BUG-002): The `--offline` flag was missing from the `tests` subcommand. Now available on all scan subcommands.
- **CORS dev-path false negatives** (BUG-003): AUTH-005 suppression for dev/test files now uses exact path segment matching instead of substring matching. Files like `dev_config.py` are still correctly identified via prefix matching (`dev_`, `test_`, etc.).
- **AWS example keys not detected** (BUG-006): Added `EXAMPLE` and `AKIA[A-Z0-9]{16}` (AWS example access key) patterns to placeholder detection. SEC-001 now catches these.
- **Path validation on scan** (BUG-007): `vigil scan nonexistent/` now returns exit code 2 with a clear error message instead of silently scanning nothing.
- **Redis connection strings** (BUG-008): SEC-003 now detects Redis password-only connection strings (`redis://:password@host`) where the username is empty.
- **Invalid YAML config handling** (BUG-009): Invalid `fail_on` values in `.vigil.yaml` now produce a clear error message and exit code 2 instead of a Pydantic validation traceback.
- **`--rule` vs `enabled: false` precedence** (BUG-012): Explicitly requesting a rule via `--rule DEP-001` now overrides `enabled: false` in config, matching the documented config precedence (CLI > YAML > defaults).
- **Similarity search performance** (BUG-013): Added length-based early rejection in `find_similar_popular()`, reducing similarity search time by ~3.4x (77s → 23s for 200 dependencies).

### Performance

- Similarity search uses only Damerau-Levenshtein (not both Levenshtein + DL), eliminating redundant computation
- Length-based early rejection skips candidates that can't possibly match the similarity threshold

## Known Limitations

- Detection is regex-based (not AST) — this is by design for V1
- Block comments (`/* */`, `"""`) not detected as comments
- SEC-005 (file not in .gitignore) not implemented — requires .gitignore parsing
- Login endpoints (POST /login) intentionally trigger AUTH-002 — disable via rule override
- Only Python and JavaScript/TypeScript supported
