# v0.1.0 — Foundation Release

> **Pre-release** | March 2026

**vigil** is a deterministic CLI static analysis tool that detects vulnerabilities specific to AI-generated code — hallucinated dependencies, insecure auth patterns, placeholder secrets, and empty test theater. No LLMs, no API costs. Complements Semgrep, Snyk, and CodeQL.

This release includes the core engine, CLI, configuration system, rule catalog, all output formatters, and the **Dependency Analyzer** — the first functional analyzer that detects hallucinated packages, typosquatting, and suspicious dependencies.

---

## Highlights

- **Full CLI** with 5 subcommands: `scan`, `deps`, `tests`, `init`, `rules`
- **Dependency Analyzer** — detects hallucinated packages (DEP-001), suspiciously new packages (DEP-002), typosquatting (DEP-003), missing source repos (DEP-005), nonexistent versions (DEP-007)
- **Registry verification** — PyPI and npm with file-based cache and offline mode
- **Typosquatting detection** — Levenshtein similarity against corpus of ~170 popular packages
- **26 security rules** defined across 4 AI-specific categories
- **4 output formats**: human (terminal), JSON, JUnit XML, SARIF 2.1.0
- **Configuration system** with YAML support, three-layer merge, and strategy presets
- **632 tests** passing — core models, engine, config, CLI, all formatters, dependency analyzer, and QA edge cases

## Installation

```bash
pip install vigil-ai-cli
```

Requires **Python 3.12+**. No external services needed.

## Usage

```bash
vigil scan src/                              # Full project scan
vigil deps --verify                          # Check dependencies
vigil tests tests/                           # Analyze test quality
vigil init --strategy standard               # Generate config
vigil rules                                  # List all 26 rules
vigil scan src/ --format sarif -o report.sarif  # SARIF for GitHub/GitLab
```

## What's Included

### Dependency Analyzer (CAT-01)

The first functional analyzer — detects AI-hallucinated dependencies, typosquatting, and suspicious packages:

- **Parsers** for `requirements.txt`, `pyproject.toml`, and `package.json` with line number tracking
- **Registry client** for PyPI and npm with file-based cache (`~/.cache/vigil/registry/`, 24h TTL)
- **Typosquatting detection** via Levenshtein distance with PEP 503 name normalization
- **Builtin corpus** of ~100 PyPI + ~70 npm popular packages (fallback until data files are generated)
- **Rules active**: DEP-001 (hallucinated), DEP-002 (new package), DEP-003 (typosquatting), DEP-005 (no source repo), DEP-007 (nonexistent version)
- **Deferred**: DEP-004 (download stats API needed), DEP-006 (import parser needed)

### Core Engine

- **ScanEngine** orchestrator — registers analyzers, collects files, runs analysis, applies rule overrides, sorts findings by severity
- **ScanResult** — aggregates findings with severity counts, threshold filtering (`findings_above`), error tracking, and timing
- **File collector** — language-aware file discovery with `os.walk` + directory pruning, always includes dependency manifests regardless of language filter
- **BaseAnalyzer Protocol** — interface contract for all analyzer implementations

### Finding Model

- **Finding** dataclass with `rule_id`, `category`, `severity`, `message`, `location`, `suggestion`, and extensible `metadata`
- **Severity** enum: `CRITICAL`, `HIGH`, `MEDIUM`, `LOW`, `INFO`
- **Category** enum: `dependency`, `auth`, `secrets`, `test-quality`
- **Location** with file, line, column, end_line, and code snippet support

### Rule Catalog (26 rules)

| Category | Rules | Description |
|---|---|---|
| **Dependency Hallucination** | DEP-001..007 | Packages that don't exist, typosquatting, suspiciously new, missing versions |
| **Auth & Permission** | AUTH-001..007 | Missing auth middleware, CORS `*`, hardcoded JWT secrets, insecure cookies |
| **Secrets & Credentials** | SEC-001..006 | Placeholder secrets, low-entropy hardcoded values, `.env.example` copies |
| **Test Quality** | TEST-001..006 | Empty asserts, trivial checks, skipped tests without reason, mock mirrors |

All rules include OWASP Top 10 for LLMs and CWE references where applicable.

### Rule Registry

- Lookup by ID, category, or severity
- Override-aware filtering (enable/disable rules, change severity via config)

### Configuration System

- **Pydantic v2 models** for validated configuration: `ScanConfig`, `DepsConfig`, `AuthConfig`, `SecretsConfig`, `TestQualityConfig`, `OutputConfig`, `RuleOverride`
- **YAML loader** with automatic discovery (`.vigil.yaml` traversing up the directory tree)
- **Three-layer merge**: defaults <- YAML file <- CLI flags
- **Strategy presets** for `vigil init`:
  - `strict` — fail on medium, 60-day min package age, 1h max token lifetime
  - `standard` — sensible defaults
  - `relaxed` — fail on critical only, 7-day min age, 72h max token lifetime

### CLI

| Command | Description |
|---|---|
| `vigil scan [PATHS]` | Full security scan with format, output, fail-on, category, and rule filtering |
| `vigil deps [PATH]` | Dependency-focused scan with `--verify/--no-verify` |
| `vigil tests [PATHS]` | Test quality analysis with `--min-assertions` |
| `vigil init [PATH]` | Config generation with `--strategy` presets and `--force` |
| `vigil rules` | Lists all rules grouped by category with severities and references |

**Key flags:** `--format` (`human`/`json`/`junit`/`sarif`), `--output`, `--fail-on`, `--category`, `--rule`/`--exclude-rule`, `--language`, `--offline`, `--changed-only`, `--verbose`, `--quiet`

**Exit codes:** `0` = clean, `1` = findings above threshold, `2` = execution error

### Output Formatters

- **Human** — terminal output with ANSI colors, severity icons, suggestions, and statistical summary
- **JSON** — structured output for programmatic consumption
- **JUnit XML** — CI dashboard integration
- **SARIF 2.1.0** — GitHub Code Scanning and GitLab SAST integration with full rule metadata

### Structured Logging

- `structlog` configured to stderr (never mixes with stdout findings output)
- Verbose mode (DEBUG) and quiet mode supported

### Git-Aware Scanning

- `--changed-only` uses `git diff --name-only HEAD` with fallback to `git status --porcelain`
- Graceful failure if git is not available
