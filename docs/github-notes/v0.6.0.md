# v0.6.0 — Integration + End-to-End Tests + QA (FASE 5)

## Changes

### CLI Improvements
- **Fixed `--changed-only`**: Now uses `git status --porcelain -u -z` for NUL-separated output, correctly handling filenames with spaces, renames, copies, and all change types (staged, unstaged, untracked)
- **Deleted files excluded**: `--changed-only` no longer includes deleted files in the scan (both staged and unstaged deletes)

### Integration Test Suite
- **71 end-to-end tests** running real analyzers against realistic AI-generated fixtures
- **Insecure project fixture**: Complete project with hallucinated deps, auth issues, hardcoded secrets, and test theater (8 files)
- **Clean project fixture**: Legitimate code verified to produce zero false positives (3 files)
- **All 4 output formats tested** with real scan results (human, JSON, SARIF, JUnit)
- **Analyzer isolation verified**: Failing analyzer doesn't crash the scan
- **Rule override integration**: disable, severity change, category filter, exclude-rule tested end-to-end

### QA Audit
- **111 QA regression tests** covering edge cases, false positives, false negatives, CLI, configuration, and integration
- **Edge cases tested**: empty files, UTF-8 BOM, Latin-1 encoding, binary files, filenames with spaces, deep nesting, nonexistent paths, duplicate paths
- **False positive verification**: clean auth/secrets/test code confirmed to produce zero findings
- **False negative verification**: insecure code confirmed to trigger specific rules (AUTH-002/004/005, SEC-001/003/004, TEST-001/002)
- **5 issues documented** with tests: `include` field unused, `_should_run()` rules_filter incomplete, YAML type safety, AUTH-005 suppression in test paths, AUTH-002 on login endpoints

### Test Coverage
- `__main__.py` now tested (`python -m vigil --help/--version`)
- `BaseAnalyzer` protocol conformance verified for all 4 analyzers
- `_get_changed_files()` unit tests: renames, copies, spaces in filenames, deleted files, staged/unstaged/untracked, truncated output, edge cases

### Fixtures
- `tests/fixtures/integration/insecure_project/` — 8 files representing a realistic AI-generated project with security issues across all 4 categories
- `tests/fixtures/integration/clean_project/` — 3 files of legitimate code for false positive verification
