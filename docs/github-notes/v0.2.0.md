# v0.2.0 — Dependency Analyzer + QA Audit

> **Pre-release** | March 2026

This release completes the **Dependency Analyzer** (FASE 1) with a full QA audit — 4 bugs fixed, 162 new tests, and comprehensive edge case coverage. The analyzer is now production-ready for detecting hallucinated packages, typosquatting, and suspicious dependencies.

---

## Highlights

- **Dependency Analyzer fully audited** — 282 tests covering all 5 active DEP rules with false positive/negative testing
- **4 bugs fixed** during QA audit (see Fixed section below)
- **632 tests** passing with ~94% coverage (up from 470 / ~92%)
- **Hardcoded version removed** from JSON formatter — now uses `__version__` consistently

## What Changed Since v0.1.0

### Added

- **162 QA tests** across 5 new test files:
  - `test_parsers_qa.py` — environment markers, BOM, CRLF, Unicode, URL deps
  - `test_registry_client_qa.py` — cache edge cases, sanitize key, response parsing
  - `test_similarity_qa.py` — corpus integrity, Levenshtein edges, PEP 503, false positive prevention
  - `test_analyzer_qa.py` — false positives/negatives, boundary conditions, metadata completeness
  - `test_integration_qa.py` — engine+analyzer integration, CLI+deps, regression tests
- **New test fixtures** in `tests/fixtures/deps/`:
  - `clean_project/` — legitimate packages (no findings expected)
  - `vulnerable_project/` — mix of legitimate and suspicious deps
  - `edge_cases/` — empty, comments-only, markers, URLs, malformed files

### Fixed

- **`--rule` flag not filtering findings** — `rules_filter` was set by CLI but `_apply_rule_overrides` never checked it; findings were returned unfiltered regardless of `--rule` flag
- **Duplicate chalk entry in npm corpus** — `similarity.py` had two entries for `"chalk"` (22M and 300K downloads); the lower value overwrote the correct one
- **Environment markers breaking requirements.txt parsing** — dependencies with markers like `pywin32; sys_platform == "win32"` were silently skipped
- **`_extract_roots` returning CWD for empty input** — when no files matched, the analyzer would scan the current working directory instead of returning no findings
- **JSON formatter hardcoded version** — replaced `"0.1.0"` string with `__version__` import for consistency with human and SARIF formatters

### Changed

- **CLI tests refactored** — `test_cli.py` and `test_cli_edge_cases.py` now use `tmp_path` with clean files instead of scanning the project root, preventing test fixture interference
