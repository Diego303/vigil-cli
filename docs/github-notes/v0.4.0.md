# v0.4.0 — Test Quality Analyzer

> **Pre-release** | March 2026

This release completes **FASE 3** with the **TestQualityAnalyzer** (TEST-001 to TEST-006). vigil now detects test theater — tests that pass but verify nothing real — in pytest (Python) and jest/mocha (JavaScript/TypeScript) codebases.

All 4 analyzers are now active. 24 of 26 rules implemented.

---

## Highlights

- **TestQualityAnalyzer** — 6 rules detecting test theater: tests without assertions, trivial assertions, catch-all exception handlers, unjustified skips, API tests without status code checks, and mock mirrors
- **1170 tests** passing with ~98% coverage (up from 961 / ~98%)
- **209 new tests** across test quality analyzer with comprehensive QA audit
- **6 bugs found and fixed** during QA audit (async tests, single-line tests, BaseException catch, quote mismatch)

## What Changed Since v0.3.0

### Added

- **TestQualityAnalyzer** (`src/vigil/analyzers/tests/`) with 4 modules:
  - `assert_checker.py` — test function extraction, assertion counting, trivial/catch-all/skip/API detection
  - `mock_checker.py` — mock return value extraction and mirror detection
  - `coverage_heuristics.py` — test file identification and framework detection
  - `analyzer.py` — orchestrates TEST-001 through TEST-006

- **Test quality rules implemented:**
  - TEST-001: Test without assertions (HIGH)
  - TEST-002: Trivial assertion — only when ALL assertions are trivial (MEDIUM)
  - TEST-003: Catch-all exceptions in tests (MEDIUM)
  - TEST-004: Skipped test without reason (LOW)
  - TEST-005: API test without status code verification (MEDIUM)
  - TEST-006: Mock mirror — mock returns same value that is asserted (MEDIUM)

- **Python support:** `def test_*`, `async def test_*`, class methods, single-line tests, pytest and unittest patterns
- **JavaScript support:** `test()`, `it()`, jest/mocha patterns, `test.skip`, `xit`, `xtest`
- **Test fixtures** in `tests/fixtures/tests/` with vulnerable and clean test samples for both Python and JavaScript
- **CLI wiring** — TestQualityAnalyzer registered in `_register_analyzers()` alongside Dependency, Auth, and Secrets analyzers
- **Configuration** — `TestQualityConfig` with `min_assertions_per_test`, `detect_trivial_asserts`, `detect_mock_mirrors`

### Fixed

- **Async test functions not detected** — regex now matches `async def test_*`
- **Single-line test assertions not counted** — `def test_x(): assert True` now correctly detected
- **`except BaseException` not detected** — catch-all regex now matches `BaseException`
- **Mismatched quotes accepted as literals** — `'hello"` no longer passes literal validation
- **JS status assertion incomplete** — `toHaveProperty('status')` now recognized
- **CLI test fixtures updated** — existing test content updated to avoid false positives from the new analyzer
