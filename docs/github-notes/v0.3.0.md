# v0.3.0 — Auth & Secrets Analyzers

> **Pre-release** | March 2026

This release completes **FASE 2** with two new analyzers: **AuthAnalyzer** (AUTH-001 to AUTH-007) and **SecretsAnalyzer** (SEC-001 to SEC-006). vigil now detects insecure auth patterns and hardcoded secrets in Python (FastAPI/Flask) and JavaScript (Express) codebases.

---

## Highlights

- **AuthAnalyzer** — 7 rules detecting insecure authentication patterns: endpoints without auth middleware, excessive JWT lifetimes, hardcoded JWT secrets, CORS allow-all, insecure cookies, and timing-unsafe password comparisons
- **SecretsAnalyzer** — 5 rules detecting hardcoded secrets: placeholder values, low-entropy secrets, connection strings with credentials, env vars with defaults, and values copied from `.env.example`
- **961 tests** passing with ~98% coverage (up from 632 / ~94%)
- **329 new tests** across auth and secrets analyzers with comprehensive QA

## What Changed Since v0.2.0

### Added

- **AuthAnalyzer** (`src/vigil/analyzers/auth/`) with 4 modules:
  - `endpoint_detector.py` — detects HTTP endpoints in FastAPI, Flask, and Express
  - `middleware_checker.py` — verifies auth middleware presence (Depends, passport, etc.)
  - `patterns.py` — regex patterns for JWT, CORS, cookies, and password comparison
  - `analyzer.py` — orchestrates AUTH-001 through AUTH-007

- **SecretsAnalyzer** (`src/vigil/analyzers/secrets/`) with 4 modules:
  - `placeholder_detector.py` — 30 regex patterns for common placeholder values
  - `entropy.py` — Shannon entropy calculation for secret strength assessment
  - `env_tracer.py` — parses `.env.example` files and traces values into source code
  - `analyzer.py` — orchestrates SEC-001 through SEC-006

- **Auth rules implemented:**
  - AUTH-001: Unprotected sensitive endpoint (HIGH)
  - AUTH-002: Destructive endpoint without authorization (HIGH)
  - AUTH-003: Excessive JWT token lifetime (MEDIUM)
  - AUTH-004: Hardcoded JWT secret with low entropy (CRITICAL)
  - AUTH-005: CORS configured with `*` allowing all origins (HIGH)
  - AUTH-006: Cookie without security flags (MEDIUM)
  - AUTH-007: Password comparison not timing-safe (MEDIUM)

- **Secrets rules implemented:**
  - SEC-001: Placeholder secret in code (CRITICAL)
  - SEC-002: Low-entropy hardcoded secret (CRITICAL)
  - SEC-003: Connection string with embedded credentials (CRITICAL)
  - SEC-004: Sensitive env var with hardcoded default (HIGH)
  - SEC-006: Value copied from `.env.example` (CRITICAL)

- **Test fixtures** in `tests/fixtures/auth/` and `tests/fixtures/secrets/` with vulnerable and clean project samples

- **CLI wiring** — both analyzers registered in `_register_analyzers()` and run automatically on `vigil scan`

### Fixed

- **Version bump** — all version references updated to 0.3.0 across source, tests, and formatters
- **Placeholder patterns expanded** — from 12 to 30 default patterns for better AI-generated placeholder detection

### Deferred

- **SEC-005** (secret file not in `.gitignore`) — requires `.gitignore` analysis, planned for a future release
