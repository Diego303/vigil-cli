# v0.5.0 — Reports + Formatters Polish

> **Pre-release** | March 2026

This release completes **FASE 4** with production-quality report formatters. All 4 output formats (human, JSON, JUnit XML, SARIF 2.1.0) have been polished, QA-audited, and regression-tested.

---

## Highlights

- **HumanFormatter** — quiet mode, configurable suggestions, ANSI color toggle, Unicode icons (✗/⚠/→/─/✓)
- **SarifFormatter** — full SARIF 2.1.0 compliance with PascalCase rule names, `ruleIndex`, `invocations`, `helpUri`, `defaultConfiguration`, OWASP/CWE in properties
- **JunitFormatter** — `<properties>` metadata element, corrected `tests` count including error testcases
- **JsonFormatter** — `summary` statistics section, conditional `snippet` in location
- **1336 tests** passing with 99% coverage on reports module (up from 1170 / ~98%)
- **166 new tests** across formatter improvements and QA regression tests
- **4 bugs found and fixed** during QA audit

## What Changed Since v0.4.0

### Added

- **HumanFormatter options** — `colors` (bool), `show_suggestions` (bool), `quiet` (bool) constructor kwargs
- **Unicode icons** — ✗ for critical/high, ⚠ for medium, → for suggestions, ─ for summary separator, ✓ for analyzers
- **Quiet mode** — suppresses header and summary, outputs only findings and errors
- **JSON summary** — `build_summary()` statistics included in JSON output
- **JUnit properties** — `<properties>` element with `vigil.version`, `vigil.files_scanned`, `vigil.analyzers`
- **SARIF improvements** — `ruleIndex`, `helpUri`, `snippet` in region, `invocations` with error notifications, `defaultConfiguration`, `semanticVersion`, OWASP/CWE in rule `properties`
- **PascalCase rule names** — `_to_pascal_case()` for SARIF rule name formatting
- **Summary by_file** — top 10 most-affected files ranking in summary stats
- **Formatter factory kwargs** — `get_formatter()` passes `**kwargs` to HumanFormatter; other formatters ignore extra kwargs
- **CLI wiring** — `vigil scan` passes `colors`, `show_suggestions`, `quiet` from config to formatter
- **166 new tests** — 77 feature tests + 89 QA regression/edge case tests

### Fixed

- **"No findings." shown with errors** — HumanFormatter displayed "No findings." even when analyzer errors existed, giving a false clean-scan impression
- **SARIF ruleIndex unsafe default** — used fallback 0 for unknown rule IDs instead of failing fast
- **SARIF double registry lookup** — redundant `registry.get()` call per rule eliminated
- **JUnit tests count** — `tests` attribute didn't count error testcases, causing mismatched totals
